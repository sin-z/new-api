# Seedance 2.0 分辨率定价配置排查

Date: 2026-07-06
Task: 回答 `new-api` 中 `seedance-2-0` 模型不同分辨率的定价如何配置
Status: completed

## 结论

- `doubao-seedance-2-0-260128` 和 `doubao-seedance-2-0-fast-260128` 的基础价格需要在后台模型价格配置中设置 `ModelRatio`。
- 不同输出分辨率和是否包含视频输入的价差不通过后台 JSON 单独配置；当前代码在 `relay/channel/task/doubao/constants.go` 的 `videoPriceTable` 中硬编码上游单价，并按 `实际单价 / 基准价` 计算 `OtherRatios["video_input"]`。
- 基准价是 `{480p/720p, 不含视频输入}`。代码注释明确该基准价等于管理员应配置的 `ModelRatio` 对应价格。

## 已核验事实

- `relay/channel/task/doubao/constants.go` 定义 Seedance 2.0 的分辨率 / 视频输入价格表。
- `relay/channel/task/doubao/adaptor.go` 的 `EstimateBilling` 从请求 metadata 读取 `resolution` 和是否含 `video_url`，返回 `video_input` 倍率。
- `relay/relay_task.go` 在任务提交前把 adaptor 返回的 `OtherRatios` 乘到基础 `Quota` 上。
- `service/task_billing.go` 在异步任务成功且返回 `totalTokens` 后，重新按 `modelRatio * groupRatio * otherMultiplier` 差额结算。
- `setting/ratio_setting/model_ratio.go` 注释定义 `ModelRatio` 单位：`1 === $0.002 / 1K tokens`，即 `$2 / 百万 tokens`。

## 配置换算

`ModelRatio = 目标单价(美元/百万 token) / 2`

若使用代码中当前人民币单价口径，需要先把人民币价换算成美元价，再按上式配置；也可以直接按业务希望的内部美元单价配置基础档，其他档位由代码中的相对倍率自动调整。
