## 目标

将 `xrtokenarkvideo` 任务状态查询从旧的 `/v1/contents/generations/tasks/{task_id}` 改为 XRToken 视频生成查询接口 `/v1/videos/generations/{task_id}`，并把上游返回转换为当前已实现的 Seedance native task 响应协议。

## Scope

- 修改 `relay/channel/task/xrtokenarkvideo` 查询路径、查询响应模型和解析逻辑。
- 保持创建任务接口不变，仍走现有 `/v1/contents/generations/tasks`。
- Seedance native GET/list 继续对外返回 public task id，不暴露上游 task id。
- 扩展 native 渲染字段，优先保留 XRToken / Seedance native 查询返回中的 `usage`、`content.video_url`、`last_frame_url`、`seed`、`resolution`、`ratio`、`duration`、`framespersecond`、`service_tier`、`execution_expires_after`、`generate_audio`、`draft`、`priority` 等字段。

- `relay/channel/task/xrtokenarkvideo/adaptor_test.go`
- `controller/seedance_native.go`
- `controller/seedance_native_test.go`

- `docs/changes.md`
- `docs/plan_index.md`

## Risks

- 查询接口路径变更只应影响 `xrtokenarkvideo` 的任务轮询，不应影响 Doubao / VolcEngine / 其他 task adaptor。
- 上游可能同时存在顶层 `video_url` 与 `content.video_url`，解析需兼容两种来源，避免已有存量任务数据无法渲染。
- `draft=false`、`priority=0` 这类新增字段零值按 `omitempty` 省略，避免为零值输出引入指针字段。
- 外部 HTTP 查询沿用现有 task adaptor `FetchTask` 接口与 `service.GetHttpClientWithProxy` 模式；本次不重构为 `manager + proxy.InitHTTP + hlhttp.NewReq + [[server_client]]`，原因是该路径属于 task adaptor 统一轮询接口，当前改动仅做协议路径和响应映射的最小修复，避免扩大渠道接入边界。

## Acceptance

- `FetchTask` 请求路径为 `/v1/videos/generations/{task_id}`。
- `ParseTaskResult` 在成功任务中写入 `TaskInfo.CompletionTokens` 和 `TaskInfo.TotalTokens`。
- Seedance native `/api/v3/contents/generations/tasks/{task_id}` 返回中包含 `usage.completion_tokens` 与 `usage.total_tokens`。
- native GET 对 XRToken 新查询响应尽量保留 Seedance 原生字段，并继续使用 public task id。
- 已补充 RED-GREEN 测试覆盖查询路径、usage 映射、native 字段渲染和存量顶层 `video_url` 兼容。
